  1. What does doctype do at the top of your html file do? Why does this need to be specified?

 A doctype is an instruction which tells the Web browser about the mark up language in which the current page is written. The doctype is not an element or tag, it lets the browser know about the version of or standard of HTML or any mark up language that is being used in the document.

2. Explain how a browser determines what HTML Elements match a CSS selector

Browsers filter out elements in the DOM according to the key selector, and traverse up its parent elements to determine matches. The shorter the length of the selector chain, the faster the browser can determine if that element matches the selector.

3. What’s the difference between and HTML Element and and HTML Tag?

An HTML element is the collection of start tag, its attributes, an end tag and everything in between. On the other hand an HTML tag (either opening or closing) is used to mark the start or end of an element

4. In your own words, explain the Cascade of CSS?

From my understanding, Cascade of CSS is specific designed structures and rules of how those elements and tags are being laid out and prioritized when there are multiple tasks performed. If we understand these structures and designs, it will be so much easier to create, develop, modify, and improve the programs.

5. Explain, to someone you know, the 3 ways to link/use CSS in an HTML file to style a web page:

  • DIRECTLY CODE INLINE TO THE HTML TAG

 Style attribute (on HTML tag) — <p style="STYLES HERE">

  • USING THE STYLE TAG

Style tag (in the head section) — <style>STYLES HERE</style>

  • LINKING TO A CSS FILE

Link tag (in the head section) — <link rel="stylesheet" href="STYLE.CSS">
